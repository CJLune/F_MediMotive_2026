## P1 CRITICAL — Fix before or at DE launch

### Issue 1 — ALL EN pages are missing hreflang

**Why this matters:** Hreflang only works when **both** pages in a pair reference each other. Every DE page already has the correct triplet (`de`, `en`, `x-default`). But no EN page has any hreflang. Google will silently ignore all DE-side hreflang signals because the EN side doesn't confirm them. Result: risk of duplicate-content penalties across EN/DE after launch, and no locale targeting.

**Affected EN pages (17):**

| EN file | Required EN canonical | Required hreflang triplet EN side |
|---------|----------------------|-----------------------------------|
| `index.html` | `/` | hreflang="en" href="https://medimotive.de/", hreflang="de" href="https://medimotive.de/de/", hreflang="x-default" href="https://medimotive.de/" |
| `about.html` | `/about` | → `/de/about` |
| `services.html` | `/services` | → `/de/services` |
| `our-approach.html` | `/our-approach` | → `/de/our-approach` |
| `case-studies.html` | `/case-studies` | → `/de/case-studies` |
| `certificates.html` | `/certificates` | → `/de/certificates` |
| `regions.html` | `/regions` | → `/de/regions` |
| `gallery.html` | `/gallery` | → `/de/gallery` *(see Issue 9 first)* |
| `contact.html` | `/contact` | → `/de/contact` |
| `imprint.html` | `/imprint` | → `/de/imprint` |
| `privacy.html` | `/privacy` | → `/de/privacy` |
| `services/early-phase-risk-design-for-quality.html` | `/services/early-phase-risk-design-for-quality` | → `/de/services/early-phase-risk-design-for-quality` |
| `services/knowledge-gap-transition-security.html` | `/services/knowledge-gap-transition-security` | → `/de/services/knowledge-gap-transition-security` |

| `services/ramp-up-process-stability.html` | `/services/ramp-up-process-stability` | → `/de/services/ramp-up-process-stability` |
| `services/rapid-response-troubleshooting.html` | `/services/rapid-response-troubleshooting` | → `/de/services/rapid-response-troubleshooting` |
| `services/supplier-quality-complaint-management.html` | `/services/supplier-quality-complaint-management` | → `/de/services/supplier-quality-complaint-management` |

**Do NOT add hreflang to these EN pages** — no DE counterpart exists:
- `work-journey.html`
- `case-coated-aluminum-parts.html`
- `case-production-ramp-up.html`

**Exact block to insert** after `<!-- seo:canonical-og -->` canonical tag on every paired EN page (substitute `{en-path}` and `{de-path}`):

```html
<link rel="alternate" hreflang="en" href="https://medimotive.de/{en-path}" />
<link rel="alternate" hreflang="de" href="https://medimotive.de/de/{de-path}" />
<link rel="alternate" hreflang="x-default" href="https://medimotive.de/{en-path}" />
```

Example for `index.html`:
```html
<link rel="alternate" hreflang="en" href="https://medimotive.de/" />
<link rel="alternate" hreflang="de" href="https://medimotive.de/de/" />
<link rel="alternate" hreflang="x-default" href="https://medimotive.de/" />
```

Example for `services/rapid-response-troubleshooting.html`:
```html
<link rel="alternate" hreflang="en" href="https://medimotive.de/services/rapid-response-troubleshooting" />
<link rel="alternate" hreflang="de" href="https://medimotive.de/de/services/rapid-response-troubleshooting" />
```

### Issue 3 — case-coated-aluminum-parts.html has a duplicate schema block

The `@graph` block (BreadcrumbList + Article) is emitted **twice** in the same `<head>`. While Google typically handles this gracefully, it will generate a validation warning in Google's Rich Results Test and is a code error.

### Issue 5 — de/contact.html ContactPage.mainEntity.url is wrong

**Current:**
```json
"mainEntity": {"@type":"Organization","name":"MediMotive","email":"info@medimotive.de","url":"https://medimotive.de/"}
```

The `url` field on the Organization points to the EN root (`https://medimotive.de/`), not the page's own entity. This is a minor schema self-reference error.

**Fix:** Update `url` to the contact page URL (or leave as the canonical Organization URL — both are acceptable, but if using the page itself, point to `https://medimotive.de/de/contact`). The simplest correct form:
```json
"mainEntity": {"@type":"Organization","name":"MediMotive","email":"info@medimotive.de","url":"https://medimotive.de/de/contact"}
```

### Issue 6 — EN privacy.html meta description contains editorial draft note

### Issue 9 — gallery.html noindex creates hreflang signal conflict

- `gallery.html` (EN): `noindex, follow`
- `de/gallery.html` (DE): has hreflang pointing to `/gallery` (EN, noindexed)

Google can still process hreflang on noindex pages, but pointing a language alternate at a noindexed URL creates a mixed signal. Two options:

**Option A (if gallery stays noindex forever):** Remove hreflang from `de/gallery.html`. Delete the three `<link rel="alternate">` tags.  
**Option B (if gallery will be indexed later):** Remove `noindex` from both gallery pages when ready.

Action required: decide which option applies, then implement. Currently leaving as-is is an accepted risk.
